# Anti-Slop — Craft Rules for Product Videos

Concrete rules that distinguish a video shipped by someone who has actually marketed a product from "default AI promo output." If a stranger can't identify which product the video is about from a random frame, you shipped a template.

Distilled from public craft guidance on AI-generated UI (anti-AI-slop conventions, animation discipline rules) and tightened for product-video use.

## P0 — Cardinal sins (don't ship)

### 1. Default Tailwind indigo / purple gradient hero

Exactly `#6366f1`, `#4f46e5`, `#4338ca`, `#3730a3`, `#8b5cf6`, `#7c3aed`, `#a855f7`. **The textbook AI tell.** Same applies to two-stop "trust gradients": purple→blue, blue→cyan, indigo→pink. A flat surface with intentional type beats this every time. Use the accent from `DESIGN.md`; if `DESIGN.md` doesn't have one, ask before inventing.

### 2. Emoji as feature icons

`✨ 🚀 🎯 ⚡ 🔥 💡` inside scene headlines, badges, or callouts. Use 1.6–1.8px-stroke monoline SVG with `currentColor`. Emoji in a 90px-headline at 1920×1080 reads as amateur theatre.

### 3. Invented metrics

"10× faster", "99.9% uptime", "3× more productive", "$2M raised". If the user hasn't provided real numbers, do not invent them. Either:
- Ask the user for the real stat, OR
- Use the storyboard's `{stat}` placeholder verbatim — the empty value forces the user to confront the missing claim before render.

A wrong stat in a promo video is a legal/reputational issue, not a design issue.

### 4. Filler copy

`lorem ipsum`, `feature one / two / three`, `placeholder text`, "Connect. Build. Scale.", "The future of X is here.", "Reimagine your workflow." If a scene needs words and you don't have specific ones, the scene is wrong — compose a different scene, don't paper over with template copy.

### 5. Sans-serif on display text when DESIGN.md binds a serif

If `DESIGN.md` declares a display serif (e.g. "Editorial New 140px" for Vercel-style minimalism), don't hardcode Inter/Roboto/system-ui at scene level. Honour the contract.

### 6. Hero scene that animates *into* a state instead of *reorienting between* states

From animation-discipline: "animate when the user is moving through space, time, or state — not to decorate, signal 'premium', or fill silence." A product video has a different remit than a UI, but the rule holds: every entrance tween should *do work* — reveal information, signal hierarchy, or transition emotion. A title scaling 0.9 → 1.0 with no second beat is decoration, not animation.

### 7. Generic narrative arc with zero variation

Promo videos default to Hook → Pain → Solution → Features → CTA. **Introduce at least one unconventional beat**: a single-quote scene held for 4s, a competitor comparison strip, an inline mini-demo, a hard cut to a stat. The template skeleton with all five beats and nothing else is the video equivalent of a Bootstrap landing page.

## P1 — Soft tells (should fix)

- **Every entrance has the same `power3.out` ease and `0.6s` duration.** Mechanical. Vary by element importance: hero text 0.7s `expo.out`, body text 0.5s `power2.out`, badges 0.4s `back.out(1.4)`. Same ease + same duration = same emotion every time, which is no emotion.
- **Every stagger is `0.08s`.** Use 0.12s for "deliberate", 0.04s for "machine-gun-fast", 0.2s for "we're slowing down here". Stagger value is rhythm.
- **More than ~12 raw hex values across the whole composition.** DESIGN.md tokens were not honoured. Pick palette colours and stick to them.
- **`var(--accent)` used 6+ times in a single scene.** Visible accent cap is 2 uses per scene; if you need more "pop", you're using the wrong colour.
- **Decorative blob / wave SVG backgrounds.** Meaningless geometry. Use a grain-overlay or a real photographic texture if you need atmosphere.
- **Every scene at 5s.** Equal-duration scenes read as a slideshow, not a video. Hook can be 3s, feature 7s, CTA 4s — varied pacing keeps attention.

## P2 — Polish tells (nice to fix)

- **Perfect symmetric layout in every scene.** Alternate density: one tight scene followed by one breathing scene reads as intentional.
- **Animation as the only signal of state change.** Always pair motion with a static affordance — colour shift, position, label. Reduced-motion users miss pure-motion changes.
- **Ambient looping motion that runs the whole video.** A subtle pulse on the brand mark is fine; a constantly-rotating background is noise.

## AI Tool Promo Specifics

If the product you're promoting *is itself an AI tool*, the standard anti-slop rules above apply double. Three additional rules:

### 1. The dogfooding loop is the strongest possible hook

If the promo was generated by the tool it's about, **say so in the first 5 seconds**. The meta-conceit ("This was made by the tool you're watching") instantly differentiates from every other AI promo, *and* it serves as live proof the tool works — the rest of the video becomes the demo by definition.

A strong dogfood hook is 8–12 words that close the loop without needing setup. Generic templates:

> *"One command. This whole video. Made by [tool]."*
> *"[Tool] wrote this script, picked the visuals, rendered the video. Watch."*
> *"You're watching [tool] talk about [tool]."*

If the tool *didn't* produce the promo, skip this rule entirely — claiming a dogfood loop you don't have is worse than any other failure on this page.

### 2. Don't narrate pain the viewer already knows

AI promos default to a "pain → solution" arc that lists generic pain points the viewer already knows: *"Hiring an agency takes weeks and thousands of dollars. Learning motion graphics takes months. Generic AI tools all look the same."* These are universal truths in the target audience. Repeating them patronizes the viewer and wastes screen time.

Replace pain-narration with **visual proof**. A terminal showing your CLI finishing, a side-by-side before/after of a real workflow, a sped-up screen recording — any of these communicates "this works" in the same runtime budget without insulting the viewer's intelligence. *Show what your tool does*, not what alternatives are bad at.

### 3. Number phases from 1 in viewer-facing content

Developers count from 0 (file `phase-0-discovery.md`, etc.). Viewers count from 1 ("step one, step two…"). When your video's visuals show a pipeline / phase list / sequence, **always number from 1** — even if the underlying code uses 0-based indices. The two conventions can coexist: file `02-pipeline.html` can render a chip labelled "Step 3."

### CTA discipline (the final scene)

The closing scene determines whether the viewer acts. Three non-negotiables:

- **Show the full repo URL** prominently. *Saying* "available on GitHub" without showing `github.com/<owner>/<repo>` on screen is a discoverability failure — the viewer should be able to pause the video and find the project.
- **Match the canonical install command exactly.** Don't invent fake arguments. If the real command is `/yourtool` with no required args, the install pill must show `/yourtool` — not `/yourtool --project=x`. A fake argument visible on screen *teaches the viewer the wrong invocation* and creates friction the first time they actually try it.
- **Don't pad commands to look "fuller".** A bare command reads more confident than a flag-stuffed one. If your canonical invocation is short, lean into the brevity. The promise of the closing scene is "this is what you type" — keep that promise literally.

## How to add soul

Aim for **~80% proven patterns + ~20% distinctive choice.** The 20% lives in:

1. **One bold visual move** — a typography choice (a display serif at 140px with -2% tracking), a single colour decision (no accent at all, just brand-white on brand-black), an unexpected proportion (the hero takes 90% of the frame instead of 60%).
2. **Voice and microcopy.** A button label that says "Start tracking" beats one that says "Get started". A CTA line that says "Read why this is the worst onboarding video on the internet" beats "Try it free for 14 days".
3. **One micro-interaction the viewer will remember** — a number that counts up and *overshoots then settles*, a screenshot that floats in then a cursor clicks a real button, a logo whose letters arrive piece-by-piece in the order someone would write them.
4. **One product-specific detail** that could only be there if you'd used the thing — a real keyboard shortcut hint, a status badge with the actual phrasing the product uses, a real error message styled as a feature.

## The screenshot test

Take a still frame from any scene in your composition. Show it to someone outside the project. If they can identify which product it's from — you have soul. If they can't — you shipped a template, and the renderer was wasted on it.
